

# Gemini Voyager 数据管理器 (Data Manager)

这是一个轻量级、无依赖的纯前端（Vanilla JS + HTML + CSS）单文件 Web 应用。专门用于读取、解析、可视化及编辑 `gemini-voyager.folders.v1` 格式的导出数据。

## ✨ 核心特性

- **📦 纯净独立**：无需 Node.js、Webpack 或任何本地服务器。双击 `index.html` 即可在任何现代浏览器中运行。
- **🌳 现代可视化视图**：
  - **左侧目录树**：支持无限层级嵌套展示，带有清晰的层级参考线。
  - **右侧网格视图**：文件与文件夹图标区分，防误触的独立外部链接跳转按钮（`↗️`）。
- **🖱️ 强大的交互系统**：
  - **丝滑拖拽 (Drag & Drop)**：支持将文件/文件夹拖入侧边栏目录或主视图文件夹，内置**防死循环**逻辑（禁止将文件夹拖入其自身或其子文件夹）。
  - **高级多选系统**：支持点击卡片或复选框多选。实现了类似操作系统的**级联选择**（向下自动勾选子内容）和**冒泡选中**（子内容全选则自动勾选父级）。
  - **右键上下文菜单**：多选后右键可呼出菜单，进行批量移动。
- **🧭 沉浸式导航**：
  - 支持“⬅️后退 / ➡️前进”的浏览历史栈。
  - 搜索模式下，底部自动显示该文件的**完整面包屑路径**。
- **🛠️ 开发者辅助**：
  - 提供元数据查看器（`{}` 按钮），点击可弹出带有语法高亮的代码块，查看选中项的底层 JSON 数据。
- **💾 无损导入/导出**：修改数据后，点击“导出”可实时生成更新后的 JSON 文件，完全兼容原插件导入标准。

------

## 🚀 快速开始

1. 将项目代码保存为 `index.html`。
2. 使用任意现代浏览器（Chrome, Edge, Firefox, Safari 等）打开该文件。
3. 点击左上角 **导入 JSON** 按钮，选择 Gemini Voyager 导出的 `.json` 文件。
4. 编辑完成后，点击 **导出 JSON** 下载最新数据。

## 🧠 数据架构与核心逻辑

本项目的一切视图和操作都基于全局变量 `store` 进行状态驱动。了解数据结构是进行二次开发的前提。

## 1. 全局状态存储 (`store`)

JavaScript

```
let store = {
    rawData: null,          // 存储导入的完整原始 JSON 数据
    currentFolderId: null,  // 当前所在文件夹ID (null 代表根目录)
    searchQuery: '',        // 当前的搜索关键词
    selectedItems: new Set() // 选中的项目集合，格式为 Set(["folder:id1", "file:id2"])
};
```

## 2. 原始 JSON 数据结构 (`store.rawData`)

该项目严格遵循 `gemini-voyager.folders.v1` 协议，将文件夹层级和文件夹内容解耦存储：

- **`data.folders` (Array)**：扁平化的文件夹数组。层级关系通过 `parentId` 维系（`null` 代表位于根目录）。
- **`data.folderContents` (Object)**：键值对结构。键为 `folderId`，值为该文件夹下的对话（文件）数组。

## 3. 核心算法说明

- **移动逻辑 (`moveItem`)**：
  - 移动文件夹：只需修改该文件夹对象的 `parentId` 字段即可，无需在数组中改变位置。
  - 移动文件：需要从原始 `folderContents[sourceId]` 数组中 `splice` 出该文件对象，然后 `push` 到 `folderContents[targetId]` 数组中。
- **多选防打平移动 (`getTopLevelSelectedItems`)**：
  - 当父文件夹和其内部文件同时被选中时，如果直接移动所有被选中的 ID，会导致子文件脱离父文件夹并与父文件夹平级。
  - 算法在打包拖拽 Payload 或右键移动前，会过滤掉“祖先已被选中”的子元素，确保只移动最顶层的目录结构。
- **双击防抖冲突**：
  - 为了防止双击卡片进入文件夹时，触发单击选中的 UI 闪烁。卡片的 `onclick` 设置了 250ms 的定时器。若 250ms 内触发了 `ondblclick`，则会立即清除定时器。
